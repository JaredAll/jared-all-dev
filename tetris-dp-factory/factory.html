In the JaredAll/Tetris implementation, the Factory design pattern is used to instantiate the different Tetris pieces. Each of the seven different piece types extend the abstract TetrisPiece class. This class is used to manipulate a Tetris piece of arbitrary type throughout the rest of the code. For almost all operations, it doesn't matter on which specific piece the operation is performed; for example, all pieces can be shifted, rotated, and dropped the same way. 


    The only important differences between pieces are the color and the configuration of the blocks. The color of the piece is handled by configurations in the TetrisComponentFactory, and the block configurations unique to each piece type are to be defined in the virtual original_block_locations() method. This method, implemented in each of the subclasses of TetrisPiece, is called in the constructor of each TetrisPiece implementation to initialize the relationship between the block locations. This relationship is preserved by the operations in the TetrisPiece super class.

    The TetrisComponentFactory is responsible for instantiating the different implementations of TetrisPiece, and does so by consulting an enum value passed in as a parameter to the build_component method. However, after piece creation, there is no need to treat the different implementations of TetrisPiece differently. So, the build_component method returns a unique_ptr<TetrisPiece>, ready to be rotated, shifted, and dropped by the player.
